# Mission

## Product

OpenAPI Studio helps small API teams turn an OpenAPI document into a dependable workflow for editing, validating, publishing, and operating an API. It must reduce specification work without hiding the underlying OpenAPI contract.

## Commercial Goal

Reach the first **$1,000 MRR** through a narrow, reliable paid product before pursuing broad legacy parity. Optimize for activation, repeated use, conversion, and retention—not feature count.

## Primary Customer

Small software teams and API-first product companies that maintain OpenAPI 3.0 or 3.1 documents and need a shared, versioned source of truth without adopting a large API platform.

## Core Outcomes

1. Import a real JSON or YAML specification and receive useful validation.
2. Edit, save, reopen, and review changes without corrupting the contract.
3. Publish accurate API documentation.
4. Collaborate safely through workspaces, roles, revisions, and GitHub.
5. Upgrade to a paid plan and receive the promised entitlements reliably.

## Product Principles

- Ship complete vertical slices; defer disconnected infrastructure and speculative abstractions.
- Preserve OpenAPI semantics and user data ahead of convenience.
- Make authorization, auditability, failure handling, and recovery part of each feature.
- Use customer evidence and measurable funnel behavior to approve roadmap expansion.
- Treat the legacy application as behavioral evidence, not an architecture template.

## Explicit Non-Goals

Until customer evidence justifies them, exclude campaigns, recipes, an extension marketplace, VS Code support, broad third-party integrations, AI coaching, push notifications, extensive administration, and proxy-platform expansion.
